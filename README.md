# prography-11th-frontend
## 🛠 1. 개발 환경 구성 및 실행 방법
### [개발 환경]
```
Framework: React (Vite)

Language: TypeScript

Library: TanStack Query (React Query), Tailwind CSS, React Router Dom
```
### [실행 방법]
Backend 서버 실행: 제공된 B/E 프로젝트를 먼저 구동합니다.

패키지 설치:

```
npm install
```
로컬 서버 실행:

Bash
npm run dev

### [커밋 메시지 템플릿 설정]
일관된 커밋 로그를 위해 아래 명령어를 실행하여 템플릿을 적용합니다.

Bash
```
git config commit.template .gitmessage
```

# 2. AI 활용 및 학습 원칙
### [사용 도구]
AI 모델: Gemini 3 Flash

### [활용 범위 및 방식]
CSS/Style: 주로 Tailwind CSS를 활용한 레이아웃 구성 및 세부 스타일링 작업에 사용했습니다.

학습 원칙: 기존 Vue.js와 JS 환경에서 작업해오다 이번에 처음으로 React와 TS를 도입했습니다. 라이브러리에 의존하기보다 직접 구현하며 익히는 것을 목표로 하였기에, 비즈니스 로직 및 컴포넌트 설계 시 AI 사용을 최소화했습니다.

# 3. 프로젝트 설계 및 회고
### [설계에 대한 생각]
API 피드백: B/E에서 제공하는 데이터 구조 중 일부 아쉬운 부분이 있었으며, 특히 필터 타입을 별도로 제공하는 API가 있었다면 클라이언트단에서 더욱 유연한 필터 UI를 구축할 수 있었을 것 같다는 생각이 들었습니다.

UX 최적화: API 호출 시 찰나의 깜빡임을 방지하기 위해 최소 800ms의 강제 지연을 주어 로딩 상태를 안정적으로 노출했습니다. 또한, 검색 버튼이나 Enter 키 입력 시에만 API를 호출하도록 최적화했습니다.

### [현재 진행 상황 및 향후 계획]
진행 지연 사유: 현재 재직 중인 회사에서 운영하는 **'서울랜드 웹앱 프로젝트'**의 마무리 QA 테스트 기간이 겹쳐 시간을 충분히 투자하지 못했습니다.

미구현 항목: 4번 출결 정보 등록/수정 기능을 현재 완성하지 못했습니다.

향후 계획: 2월 말 회사 업무가 마무리되면, 3월부터는 많은 시간을 투자할 수 있습니다. 미흡한 부분(등록/수정 등)은 업무 종료 후 추가 커밋을 통해 반드시 완성할 예정입니다.
